Deep Dive: Der SELECT-Befehl

Daten in einer Datenbank zu speichern ist nur die halbe Miete. Die wahre Stärke einer Datenbank ist es, genau die Daten extrem schnell wiederzufinden, die man gerade braucht. Dafür nutzen wir die Sprache SQL und ihren wichtigsten Befehl: SELECT.

Die Grundstruktur (Die "Satzbausteine")

Ein SQL-Befehl liest sich fast wie ein englischer Satz. Man kann ihn durch verschiedene Bausteine (Klauseln) erweitern:

  • SELECT Spaltenname(n)
    (Pflicht! Welche Infos willst du sehen?)
  • FROM Tabellenname
    (Pflicht! Aus welchem "Aktenschrank" kommen die Daten?)
  • WHERE Bedingung
    (Optional: Filtere die Daten, z.B. Preis < 20)
  • ORDER BY Spaltenname
    (Optional: Sortiere das Ergebnis alphabetisch oder numerisch)
  • LIMIT Zahl
    (Optional: Zeige nur die ersten X Ergebnisse an)

Unsere Trainings-Datenbank

Für die folgenden interaktiven Aufgaben sind wir die Datenbank-Admins eines Elektronik-Shops. Unsere Tabelle heißt produkte. Hier ist der aktuelle Inhalt der Tabelle:

Level 1: Alles oder Nichts Anfänger

Der einfachste SELECT-Befehl nutzt einen Stern (*). Der Stern bedeutet einfach: "Zeige mir alle Spalten!".

SELECT * FROM produkte;

Wenn du nicht alle Spalten brauchst, zählst du sie einfach mit einem Komma getrennt auf:

SELECT name, preis FROM produkte;

Level 2: Die Filter-Magie Fortgeschritten

Jetzt wird es interessant. Mit WHERE können wir genaue Kriterien festlegen. Dafür nutzen wir Operatoren wie = (gleich), < (kleiner), > (größer) oder <= (kleiner gleich).

Achtung: Text muss in SQL immer in einfachen Anführungszeichen stehen! (z.B. 'Gaming'). Zahlen schreibt man ohne Anführungszeichen.

SELECT * FROM produkte WHERE kategorie = 'Audio';
SELECT name FROM produkte WHERE preis > 100;

Deine Aufgabe:

Unser Chef möchte wissen, welche Artikel fast ausverkauft sind. Schreibe einen Befehl, der alle Spalten (*) der Tabelle anzeigt, aber nur für Produkte, deren lagerbestand kleiner als 10 ist.

SQL>
✅ Perfekt! Der WHERE-Filter funktioniert.

Level 3: Sortieren und Begrenzen Profi

Wenn wir hunderte Produkte haben, wollen wir das Ergebnis sortieren. Das machen wir mit ORDER BY. Standardmäßig wird aufsteigend sortiert (ASC). Wenn du absteigend sortieren willst, hängst du ein DESC an.

Oft wollen wir auch nur die "Top 3" sehen. Dafür gibt es ganz am Ende den Befehl LIMIT.

/* Zeigt die 2 teuersten Produkte an */
SELECT name, preis FROM produkte ORDER BY preis DESC LIMIT 2;

Deine Aufgabe:

Schreibe einen Befehl, der alle Spalten (*) anzeigt. Sortiere die Produkte nach dem preis absteigend (das Teuerste zuerst). Zeige nur die ersten 3 Ergebnisse an (LIMIT).
