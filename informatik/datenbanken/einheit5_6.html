3. Die erste Datenbank & SQL

Wir haben unsere Tabellen auf Papier gezeichnet. Wenn wir diese in den Computer übertragen, müssen wir dem System sagen, welche Art von Daten in eine Spalte kommt. Ein Name ist schließlich Text, ein Preis ist eine Zahl.

Die wichtigsten Datentypen

  • INTEGER: Ganze Zahlen (z.B. für IDs oder Stückzahlen).
  • TEXT / VARCHAR: Zeichenketten, also Text (z.B. Namen, Adressen).
  • REAL / DECIMAL: Kommazahlen (z.B. für Preise oder die MwSt).

Wie spricht man mit einer Datenbank?

Um Daten abzufragen, brauchen wir eine Sprache, die die Datenbank versteht. Diese Sprache heißt SQL (Structured Query Language). Der wichtigste Befehl, um Daten auszulesen, ist der SELECT-Befehl.

Der Bauplan einer SELECT-Abfrage

Eine Abfrage ist fast wie ein normaler englischer Satz aufgebaut:

  • SELECT (Zeige mir) -> Hier sagst du, welche Spalten du sehen willst. Ein * bedeutet "alle Spalten".
  • FROM (Aus der Tabelle) -> Hier nennst du den Namen der Tabelle.
  • WHERE (Wo / Unter der Bedingung) -> Hier filterst du die Daten (z.B. Preis < 50).
  • ORDER BY (Sortiert nach) -> Hier legst du fest, ob aufsteigend oder absteigend sortiert wird.
